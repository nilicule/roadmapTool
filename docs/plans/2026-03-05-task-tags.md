# Task Tags Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add free-form string tags to tasks ‚Äî stored in YAML, editable via comma-separated input in modals, and displayed as small colored squares on the right side of task bars.

**Architecture:** Tags are a `list[str]` on the `Task` model. The API's `TaskCreate`/`TaskUpdate` gain the same field. The frontend renders one 10√ó10px colored square per tag in `renderTask`, positioned to the left of the assignee initial; `tagColor(tag)` derives a consistent hue by hashing the tag name.

**Tech Stack:** Python/Pydantic v2 (backend), FastAPI (API), vanilla JS + SVG (frontend), pytest (tests).

## Task 4: Render tag squares on task bars

**Files:**
- Modify: `static/app.js` ‚Äî add `tagColor()` helper, modify `renderTask` (line 293)

**Step 1: Add `tagColor` helper**

Add near the date utilities section (after `today()`, around line 38):

```javascript
function tagColor(tag) {
  let h = 0;
  for (let i = 0; i < tag.length; i++) h = (h * 31 + tag.charCodeAt(i)) % 360;
  return `hsl(${h}, 65%, 48%)`;
}
```

**Step 2: Add tag squares to `renderTask`**

In `renderTask`, after the assignee initials block (after line 363, before the resize handles), add:

```javascript
// Tag squares (left of assignee, right side of bar)
const tags = t.tags || [];
if (tags.length > 0 && barW > 20) {
  const SQ = 10, GAP = 4;
  const assigneeW = (t.assignee && barW > 50) ? 22 : 6;
  const rightEdge = barX + barW - assigneeW;
  tags.forEach((tag, i) => {
    const sqX = rightEdge - (i + 1) * (SQ + GAP) + GAP;
    if (sqX < barX + 4) return; // skip if no room
    svg.appendChild(svgEl('rect', {
      x: sqX, y: barY + (barH - SQ) / 2,
      width: SQ, height: SQ, rx: 2,
      fill: tagColor(tag), 'pointer-events': 'none'
    }));
  });
}
```
